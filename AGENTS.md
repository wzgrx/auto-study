# Auto-Study 项目开发指南

## 给 AI 助手的指令

### 开工前必读

每次开始新任务、修改代码、或解决问题前，**先读本项目根目录下所有 `.md` 文件**，包括但不限于：
- `ARCHITECTURE.md` — 整体架构设计
- `PLATFORM_ANALYSIS.md` — 各平台分析
- `README.md` — 项目说明

### 遇到问题优先搜索

当遇到以下情况时，**必须先搜索再动手**：

- 不确定某个功能怎么实现
- 遇到奇怪的报错
- 需要适配新的网站
- 需要新增模块
- 用户的问题涉及不熟悉的领域

**搜索优先级：**

```
1. 网络搜索（Google/Bing）— 搜方案
2. GitHub Issues / PR — 搜同类问题
3. GitHub 搜索仓库 — 搜现成实现
4. 官方文档 — 确认用法
5. 读相关模块的源码 — 逐行读
```

**原则：** 宁可一次慢，不要无头苍蝇瞎试。猜了 2 次还没解决 → 立刻搜索。

### 不重复造轮子

- GitHub 上有现成的代码 → 读懂了再用
- 有可复用的 JS 脚本 → 提取核心逻辑
- 有成熟的库 → 用库，不自己写

### 步进式执行

用户说"我说一步操作，你操作一步"时：

```
1. 等用户指令
2. 执行
3. 报告结果 + 当前状态
4. 等下一步指令
```

不猜测下一步，不提前执行，不跳过步骤。

### 模块隔离

- 模块之间不能互相调用（只能依赖 `brain.py`）
- 改一个模块不影响其他模块
- `brain.py` 的接口要稳定，改了要检查所有依赖

### 构建顺序

严格按模块编号从 1 到 18 依次构建：

```
阶段1（基础）：1 config → 2 tab_manager → 3 logger → 4 exceptions → 5 brain
阶段2（支撑）：6 anti_detect → 7 persistence → 8 captcha
阶段3（业务）：9 login → 10 scanner → 11 player → 12 quiz
阶段4（扩展）：13 notifier → 14 health → 15 plugin
阶段5（收尾）：16 progress → 17 reporter → 18 scheduler
```

每个模块写完了测试通过了再写下一个。
