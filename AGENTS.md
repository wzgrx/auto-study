# Auto-Study 项目开发指南

## ⚠️ 铁律（不遵守 = 写垃圾代码）

### 铁律 1：每次动代码前先读 AGENTS.md

每次开始任何任务前，**必须先用 read_file 重读本文件**。
不读 = 不知道规则 = 写错代码 = 用户愤怒。

### 铁律 2：先搜索再写代码

**搜索顺序（必须全部执行，缺一不可）：**
```
1. 网络搜索（搜方案）
2. GitHub 搜索（搜同类项目/代码）
3. 读官方文档（确认 API 用法）
4. 读项目内相关源码（逐行）
```

搜不到方案才允许自己写。写了发现搜到了 → 删掉改用现成的。

### 铁律 3：猜 2 次没解决 → 立刻搜索

```
第1次猜 → 失败
第2次猜 → 失败
→ 搜索！绝不猜第3次！
```

### 铁律 4：步进式执行

用户说"我说一步你操作一步"时：
```
1. 等指令
2. 执行
3. 报告结果
4. 等下一步
```

不猜测、不提前、不跳步。

### 铁律 5：模块隔离

- 模块只依赖 `brain.py` 或 `config.py`，不互相调用
- 改 brain.py 的接口要检查所有依赖模块
- 不改其他模块的代码

### 铁律 6：按编号构建

```
1→2→...→18，写一个测试一个，不跳步
```

## 历史教训（必须记住，否则重犯）

| 错误 | 教训 |
|------|------|
| tab_manager 没搜索 PyDoll/BrowserStack 就直接写 | 先搜索再看代码再写 |
| JS fallback 用 f-string 嵌套引号 | 用 json.dumps 序列化 |
| `ensure_min_pool` 只统计池内标签页 | TabManager 只复用，不新建 |
| 配置写死 `input[type='text']` | 先 inspect 页面实际 DOM |
| 验证码失败不重试 | 提供 retry 机制 |
